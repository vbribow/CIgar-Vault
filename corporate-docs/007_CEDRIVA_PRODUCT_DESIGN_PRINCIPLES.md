# Document 007 — Cedriva Product Design Principles

**Version:** 1.0  
**Status:** Approved Draft  
**Owner:** Brian Bowers  
**Created:** July 23, 2026  
**Last Updated:** July 23, 2026

Every screen should answer one question.

Every page should have one primary action.

Information should be progressive.

Complexity should reveal itself only when needed.

Advanced collectors should never be limited.

New collectors should never feel overwhelmed.

Every page should teach something.

Every cigar should tell a story.

Every collection should feel valuable.

Every collector should feel respected.

Speed matters.

Beauty matters.

Accuracy matters.

Trust matters more than everything else.

If there is a conflict between simplicity and trust:

**Trust wins.**

If there is a conflict between monetization and trust:

**Trust wins.**

If there is a conflict between growth and trust:

**Trust wins.**
